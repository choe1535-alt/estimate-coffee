# Gemini CLI 진입 가이드

이 프로젝트의 공통 LLM 가이드는 `AGENTS.md`에 정리되어 있습니다.
작업을 시작하기 전에 반드시 `AGENTS.md`를 먼저 읽고 그 규칙을 따르세요.

핵심만 요약하면:

1. 사용자는 기본적으로 비개발자입니다. 항상 친근한 한국어로 자세히 설명하고, 어려운 에러는 사람 말로 풀어서 알려주세요.
2. 세션 시작 또는 새 작업 시작 때마다 Git 정리를 제안하세요.
3. 코드 변경, 문서 수정, 배포 작업 전에는 반드시 `git fetch origin`과 `git status`를 실행하세요.
4. 로컬 수정 파일이 있으면 되돌리지 말고, 사용자에게 어떤 파일이 수정 중인지 설명한 뒤 진행 방향을 확인하세요.
5. 특이사항이 있으면 Slack DM으로 `최형림`에게 알려주세요. 확인된 Slack 사용자 ID는 `U0B1CUTTALW`입니다.
6. 다음 세션에서도 알아야 할 특이사항은 `AGENTS.md`에 기록하세요.
7. 작업이 끝나면 로컬 확인 방법과 "배포해줘/OK" 안내를 포함하세요.
8. 사용자가 "OK", "배포", "반영해줘"라고 하면 관련 파일만 커밋하고 `main`에 push한 뒤 GitHub Pages 배포 상태를 확인하세요.
9. `git reset --hard`, `git push --force`, `git clean -fd`, `rm -rf` 같은 파괴적 명령은 사용자 동의 없이 실행하지 마세요.

현재 알려진 특이사항도 `AGENTS.md`의 "현재 인지 중인 특이사항" 섹션을 확인하세요.
