# 이 프로젝트와 함께 일하는 LLM에게

이 문서는 Claude Code · Gemini CLI · Codex 등 **모든 LLM 에이전트가 이 저장소에서 작업할 때 따라야 하는 공통 규칙**입니다.
가장 먼저 읽고 시작하세요.

---

## 1. 이 프로젝트의 사용자 가정 — **기본값: 비개발자**

> 이 저장소는 **비개발자가 LLM의 도움을 받아 직접 코드를 고치고 배포하는 것**을 전제로 만들어졌습니다.
> 사용자가 명시적으로 "개발자입니다" / "제가 직접 코드를 보겠습니다" 같은 표현으로 본인을 개발자라고 밝히기 전까지는 **항상 비개발자 모드**로 응답하세요.

### 비개발자 모드 (기본)

- 응답은 친근한 한국어로, 어려운 용어는 풀어서 설명합니다.
- 에러 메시지는 그대로 던지지 말고 사람 말로 번역해서 알려줍니다.
  - `ENOENT: no such file` → "필요한 파일이 없어요. 제가 찾아서 만들게요."
  - 타입 에러 → "값 형식이 안 맞는 부분이 있어요. 제가 고칠게요."
- git 명령은 사용자가 직접 칠 필요 없도록 LLM이 알아서 실행합니다.
  - 단, **파괴적 명령**(아래 §6 참고)은 반드시 사용자 동의를 얻고 실행.
- 한 단계가 끝날 때마다 "지금까지 한 것 / 다음에 할 것"을 한두 줄로 요약합니다.

### 개발자 모드 (사용자가 본인을 개발자라고 밝힌 경우)

- 평소 LLM 톤으로 간결하게 진행합니다.
- 아래 절차 안내문(§3, §4)을 매번 띄울 필요 없습니다.
- 단, **§6 파괴적 명령 규칙**은 그대로 유효합니다.

## 2. 작업을 시작하기 전에 — git 상태 확인 (필수)

LLM은 사용자가 코드 변경/기능 추가/버그 수정 등 **무언가를 만들거나 고치는 작업을 부탁할 때마다**, 손대기 전에 반드시 아래를 먼저 실행합니다.

```bash
git fetch origin
git status
```

그 결과로:

- **원격에 새 커밋이 있는 경우** → 사용자에게 이렇게 알립니다.
  > "원격(GitHub)에 새로 올라온 변경이 있어요. 충돌을 막기 위해 먼저 `git pull` 부터 진행해도 될까요?"
  사용자가 OK 하면 `git pull --rebase origin main` 실행.

- **로컬에 안 커밋된 변경이 있는 경우** → 사용자에게 이렇게 알립니다.
  > "지금 저장은 안 했지만 수정 중인 파일들이 있어요: [파일 목록]. 이걸 마저 정리하고 작업할까요, 아니면 그대로 위에 이어서 작업할까요?"

- **깨끗한 상태** → 바로 작업 시작.

> 단순히 "이 코드 설명해줘" 같은 **읽기 전용 질문**에서는 위 절차 생략 가능.

## 3. 작업 도중

- 코드는 자유롭게 고쳐도 됩니다.
- 외부 라이브러리 추가가 필요하면 사용자에게 "OO 라이브러리를 새로 깔게요" 라고 한 줄 알린 뒤 진행.
- 빌드/타입 에러가 나면 **사용자에게 떠넘기지 말고 LLM이 끝까지 고친 뒤** 결과를 알립니다.

로컬에서 미리 확인하시려면:

   npm run dev

   👉 http://localhost:5173/estimate-coffee/

바로 실제 사이트에 반영해 드릴게요.
수정이 필요하시면 어디가 어떻게 다른지 알려주시면 됩니다.

## 5. 사용자가 "OK / 배포 / 반영해줘" 라고 답한 경우 — 배포 흐름

다음 순서대로 LLM이 자동 실행하세요.

1. `git status` 로 변경 사항 확인.
2. `git add` 로 관련 파일만 스테이지.
3. `git commit -m "<한국어 요약>"` — 평이한 한국어로 짧게 (예: `feat: 견적서 인쇄 색상 개선`, `fix: 고객 정보 기본값 제거`).
4. `git push origin main`
5. 배포가 진행되는 동안 사용자에게 다음과 같이 알립니다.

```
🚀 GitHub에 올렸어요! 배포가 진행되고 있어요.
1~2분 정도 걸려요. 다 끝나면 알려드릴게요.
```

6. `gh run watch` 가 가능하면 사용, 안 되면 `gh run list --branch main --limit 1` 폴링.
7. 배포가 끝나면 사용자에게 최종 안내.

```
🎉 배포 완료!

사이트가 새 버전으로 바뀌었어요:
👉 https://choe1535-alt.github.io/estimate-coffee/

브라우저에서 새로고침(Cmd+Shift+R)을 한 번 해주시면 확실히 보여요.
```

## 6. 파괴적 명령 — 사용자 동의 없이는 금지

다음은 **반드시 사용자에게 한 번 더 확인**한 뒤에 실행하세요. 무심코 누르면 작업물이 사라집니다.

- `git reset --hard`
- `git push --force` / `git push -f`
- `git clean -fd`
- `git branch -D`
- `rm -rf` (대상이 `node_modules`, `dist`, `.vite` 같은 캐시면 예외)
- `package-lock.json` 삭제 후 재설치

## 7. 이 프로젝트의 기술 메모 (LLM 참고용)

- **스택**: React 18 + TypeScript + Vite / React Router v7 / TanStack Query 5 / axios / zustand / Tailwind CSS / shadcn-ui 스타일 Radix Primitives.
- **데이터**: `public/api/*.json` 에 mock API. axios + TanStack Query 로 호출. 추후 사내 DB API 로 갈아끼울 자리.
- **상태**: `src/stores/quoteStore.ts` (zustand persist, key=`coffee24-quote`).
  - 기본값/스키마를 바꿨다면 `persist.version` 을 반드시 +1 해주세요. 안 그러면 사용자 브라우저에 옛 데이터가 남아 새 기본값이 안 보입니다.
- **테마**: 에디터 크롬은 **Modern Blue 고정**. 테마 스위처는 견적서 paper(`<article class="quote-paper">`)에만 `data-theme` 으로 적용됩니다. 에디터 전체에 테마가 번지면 버그입니다.
- **견적서 핵심 룰**:
  - `원두포함여부 = ISBLANK(원두) ? "X" : "O"`
  - 원두 미포함 → **머신 단독 렌탈** 섹션만 노출
  - 원두 포함 → **머신렌탈 & 원두구독** 섹션만 노출
  - 두 섹션이 동시에 보이면 룰 위반.
- **인쇄/PDF**: `src/styles/globals.css` 의 `@media print` 블록 참고. `print-color-adjust: exact` 가 빠지면 색이 다 빠집니다.

## 8. 자주 쓰는 명령

| 작업 | 명령어 |
|---|---|
| 의존성 설치 | `npm install` |
| 개발 서버 | `npm run dev` → http://localhost:5173/estimate-coffee/ |
| 프로덕션 빌드 | `npm run build` |
| 빌드 결과 미리보기 | `npm run preview` |
| 엑셀 → JSON 재생성 | `npm run build-data` (== `python3 tools/build_data.py`) |
| 배포 (자동) | `git push origin main` |
